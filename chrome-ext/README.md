# Hedgehog Vision — Chrome Extension

Auto-injects the HC_* clicker into the Ёжики game iframe so you don't have to paste it into DevTools every session.

## Install (one-time)

1. Run `node build.js` in the project root to generate `iframe-script.js`.
2. Open `chrome://extensions` in Chrome.
3. Enable **Developer mode** (top-right toggle).
4. Click **Load unpacked** and select this `chrome-ext/` folder.
5. Open `https://vk.com/ezhiky_game`. The HC panel should appear automatically.

After editing source files, run `node build.js` and click the **Reload** button next to the extension card.

## Programmatic control (postMessage bridge)

The extension listens for commands posted to the iframe's window. From the parent page (`vk.com`) JS console:

```js
const iframe = document.querySelector('iframe[src*="valley.redspell"]');
const replies = {};
window.addEventListener('message', e => {
  if (e.data?.type === 'HC_RES') replies[e.data.id] = e.data;
});

function send(cmd, ...args) {
  const id = Math.random().toString(36).slice(2);
  iframe.contentWindow.postMessage({type: 'HC_CMD', id, cmd, args}, '*');
  return new Promise(res => {
    const t = setInterval(() => {
      if (replies[id]) { clearInterval(t); res(replies[id]); }
    }, 50);
  });
}

// Usage:
await send('ping');                          // {ok, value: {canvas: [w,h]}}
await send('visitStart');
await send('visitStats');                    // {visited, cycleClicks, running}
await send('setCfg', {minCluster: 20});
await send('sample', 500, 350);              // {h, s, l, r, g, b}
await send('setHomeBtn', 80, 660);
await send('stop');
```

## Available commands

| cmd            | args                  | returns                        |
|----------------|-----------------------|--------------------------------|
| `ping`         | —                     | `{ok, canvas:[w,h]}`           |
| `start`        | —                     | clicker stats                  |
| `stop`         | —                     | clicker stats                  |
| `setMode`      | `'smart'|'grid'|'single'` | current mode               |
| `scan`         | —                     | `'scan triggered'`             |
| `getStats`     | —                     | `{clicks, scans, found, ...}`  |
| `visitStart`   | —                     | visit stats                    |
| `visitStop`    | —                     | visit stats                    |
| `visitStats`   | —                     | `{visited, cycleClicks, ...}`  |
| `setHomeBtn`   | `x, y` (canvas px)    | `{home, voyage}`               |
| `setVoyageBtn` | `x, y` (canvas px)    | `{home, voyage}`               |
| `sample`       | `x, y` (canvas px)    | `{h, s, l, r, g, b}`           |
| `getCfg`       | —                     | `HC_CFG` object                |
| `setCfg`       | `{partial cfg}`       | merged `HC_CFG`                |

## Toolbar action — open game with last captured URL

Clicking the extension icon opens the game using the most recently
captured VK launch URL. The flow:

1. First click → no URL stored, opens `https://vk.com/ezhiky_game`.
2. Sign into VK if needed; click the launch button (or let
   `vk-launcher.js` auto-click it).
3. When VK mounts the game iframe, the launcher saves
   `iframe.src` to `chrome.storage.local.lastGameUrl`.
4. Next click → opens the captured URL directly. Stays valid until
   `expire=` lapses; once expired, the icon transparently re-opens
   the VK launcher to refresh the token.

This sidesteps the long-URL-with-tokens copy/paste workflow.

## Architecture

- `manifest.json` — MV3.
  - Content scripts on `valley.redspell.ru/*` (`all_frames: true`,
    `world: "MAIN"` for `iframe-script.js`; ISOLATED for
    `iframe-isolated.js`).
  - Content script on `vk.com/ezhiky_game*` and `vk.com/app*` for
    `vk-launcher.js`.
- `iframe-script.js` — generated by `build.js`. Contains the entire
  `HC_*` stack plus a `postMessage` bridge.
- `iframe-isolated.js` — bridge between MAIN-world iframe-script and the
  background service worker (debugger clicks).
- `vk-launcher.js` — captures fresh game URLs by watching for the game
  iframe to mount on the VK app page; writes to `chrome.storage.local`.
- `background.js` — owns the `chrome.debugger` session for trusted
  clicks AND handles the toolbar action (opens stored game URL or
  routes to the VK launcher).

## Troubleshooting

- **No panel appears**: open the iframe DevTools (right-click game → Inspect → switch context to `valley.redspell.ru`) and check console for `[HC-Ext]` logs.
- **"No canvas after 60s"**: game didn't load. Refresh page.
- **Bridge doesn't respond**: confirm you're posting to `iframe.contentWindow` and listening on `window` for `HC_RES`. Origin is `*`.
